# 기능 요약

이 문서는 유지보수 관점에서 **현재 구현된 기능**과 **실제로 남아 있는 보완 과제**를 빠르게 확인하기 위한 상태 문서입니다.

## 개발된 기능

### 1. 인증 및 라우팅
- 이메일 회원가입, 로그인, 로그아웃 API와 화면이 구현되어 있습니다.
- 로그인 성공 시 JWT가 `accessToken` 쿠키로 발급되며, `/api/auth/me`로 인증 상태를 복원합니다.
- `ProtectedRoute`, `PublicRoute`로 로그인/회원가입 화면과 보호 화면(`/`, `/mypage`, `/ranking`)을 분리합니다.
- Google OAuth2 로그인 성공 시 사용자 정보를 동기화하고 동일한 JWT 쿠키를 발급합니다.

### 2. 오늘의 퀴즈 및 투표
- 오늘의 퀴즈 조회 API(`/api/quiz/today`)와 홈 화면이 구현되어 있습니다.
- 사용자는 하루 1회 상승/하락 투표를 할 수 있습니다.
- 투표 시 중복 투표, 인증 여부, 투표 가능 시간(09:00~20:00), 퀴즈 상태를 검증합니다.
- 투표 여부 확인 API(`/api/quiz/{quizId}/check-vote`)와 투표 비율 API(`/api/quiz/{quizId}/stats`)가 구현되어 있습니다.
- 홈 화면에서 현재가/종가 표시 전환, 참여 수, 상승/하락 비율, 마감 안내를 제공합니다.

### 3. 포인트, 마이페이지, 랭킹
- 마이페이지에서 닉네임, 이메일, 누적 포인트, 총 투표 수, 적중률을 조회할 수 있습니다.
- 투표 히스토리 API와 화면이 있으며 정산 대기/완료 상태, 예측 결과, 획득 포인트를 확인할 수 있습니다.
- 랭킹 API(`/api/ranking/top`, `/api/ranking/me`)와 랭킹 화면이 구현되어 있습니다.
- 정산 결과에 따라 포인트가 반영됩니다. 적중 시 2점, 보합 시 1점, 미적중 시 0점입니다.

### 4. 배치 및 스케줄링
- 매일 00:01에 오늘의 퀴즈를 자동 생성합니다.
- 매일 23:59에 오늘 퀴즈를 `CLOSED` 상태로 마감합니다.
- 다음 거래일 20:05에 이전 영업일의 `OPEN`/`CLOSED` 퀴즈를 정산하여 `SETTLED`로 변경합니다.
- 정산 시 `finalPrice`, `quizResult`, `settledAt`, `vote.isCorrect`, `vote.pointsEarned`, `user.points`를 함께 반영합니다.
- 주말과 휴장일에는 퀴즈 생성/마감/정산을 건너뜁니다.
- 주말·휴장일로 정산이 미뤄진 퀴즈는 다음 거래일 정산 시점에 함께 처리됩니다.

### 5. 로컬 수동 테스트 API
- `local` 프로필에서만 로컬 수동 테스트용 스케줄 API가 열립니다.
- 지원 API
  - `POST /api/local/schedules/create-quiz`
  - `POST /api/local/schedules/close-quiz`
  - `POST /api/local/schedules/settle`
  - `POST /api/local/schedules/settle-today`

## 추가/보완이 필요한 기능

### 1. 운영 경로
- 활성 종목이 없으면 퀴즈 생성이 중단되지만, 현재 코드 기준으로 종목/퀴즈를 운영자가 관리하는 별도 운영 화면이나 API는 없습니다.

### 2. 소셜 로그인 정리 및 확장
- 백엔드 OAuth2 사용자 처리 로직은 현재 Google만 허용합니다.
- 프론트엔드에는 Kakao 로그인 버튼과 Kakao/Naver용 URL 설정 키가 남아 있어, 실제 지원 범위와 UI/설정을 맞추는 정리가 필요합니다.

### 3. 시간/정책 정리
- 현재 코드에는 투표 가능 시간(09:00~20:00), 퀴즈 자동 마감 시간(23:59), 자동 정산 시간(다음 거래일 20:05)이 각각 따로 존재합니다.
- 운영 기준에서 이 시간 정책을 문서와 설정 관점으로 더 명확히 맞춰 둘 필요가 있습니다.

## 참고 상태

- 퀴즈 상태 전이: `OPEN -> CLOSED -> SETTLED`
- 주요 사용자 화면: 로그인, 회원가입, 홈, 마이페이지, 랭킹
