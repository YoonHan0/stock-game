# 기능 요약

## 1) 현재 구현된 기능
- 이메일 회원가입/로그인, 로그아웃, JWT 쿠키 기반 인증 복원
- 보호 라우팅 기반 로그인 화면/회원가입 화면/홈 화면 분리
- 오늘의 퀴즈 1건 조회와 현재가 표시
- 하루 1회 상승/하락 투표, 중복 투표·투표 가능 시간·마감 상태 검증
- 투표 전후 실시간 참여 수와 상승/하락 비율 조회
- 서버 시작 시 활성 종목 중 1개를 골라 오늘 퀴즈 자동 생성
- Google OAuth2 로그인 후 사용자 동기화와 JWT 쿠키 발급 구조

## 2) 앞으로 구현되어야 할 기능
- 포인트 시스템과 마이페이지 화면
- 랭킹/적중률 통계 화면 및 관련 API
- 종가 기준 자동 정산 배치
- 휴장일 대응 스케줄링
- 소셜 로그인 제공자 확장 및 정리
- 카카오 OAuth2 로그인, 회원가입 기능 구현

## 3) 현재 로직상 꼭 필요한데 아직 비어 있거나 보완이 필요한 기능
- 퀴즈 마감 후 `final_price`, `status`, `Vote.is_correct`를 실제로 갱신하는 정산 로직
    - `final_price` 값은 국내장(KRX, NXT 둘 다)이 마감되는 오후 8시에 해당 종목의 가격이 삽입된다.
    - 국내장이 마감되어 `final_price` 값이 삽입된 직후, 전날 퀴즈가 존재한다면 해당 퀴즈의 종목의 현재가(=마감가격)을 스크랩핑하여 가격이 상승/하락 여부를 확인하고, 사용자가 투표한 답을 확인하여 `vote.is_correct` 값을 업데이트한다.
    - 정산된 퀴즈의 정답/오답을 처리하고 `vote.user_id` 값을 확인하여(`vote.user_id` = `user.id`) 해당 유저의 포인트를 계산한다.(`user.points`) 정답일 경우 +2, 가격 변동이 없어서 전날 종가와 금일 종가가 동일할 경우 +1, 오답일 경우 +0으로 계산한다.
    - 국내장이 마감된 후에는 화면에 '현재가: {실제가격}원'이 아닌 '종가: {실제가격}원'으로 표기된다.
    - 퀴즈 마감은 당일 오후 11시 59분에 마감 처리를 한다.
    - 오전 12시 1분에는 새로운 퀴즈가 등록된다.
    - 
- 정답 반영 후 `User.points`를 적립하는 포인트 연결 로직
- 휴장일·비거래일에 퀴즈 생성/정산을 멈추거나 이월하는 처리
- 활성 종목이 없을 때 퀴즈가 생성되지 않으므로 종목/퀴즈를 관리할 운영 경로
- 프론트 소셜 로그인 URL, 백엔드 허용 경로, 실제 지원 provider(현재 Google 중심) 일관성 점검
